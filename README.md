El proyecto Restaurante.java desarrolla un sistema orientado a objetos para la gestión integral de un restaurante, aplicando principios de programación modular y encapsulación. La clase principal `Restaurante` implementa la interfaz `IRestaurante` y la interfaz `Serializable`, permitiendo tanto la manipulación de datos como su persistencia en memoria.
El sistema gestiona tres listas principales: **ingredientes**, **platos** y **órdenes**, representadas mediante estructuras `ArrayList`. El constructor inicializa estas colecciones vacías para permitir el registro dinámico de elementos a medida que se interactúa con el sistema.

En cuanto a la funcionalidad, el programa permite **añadir, buscar y validar ingredientes y platos**. Mediante métodos como `nuevoIngrediente` y `nuevoPlato`, se controlan las inserciones evitando duplicados, lanzando excepciones (`ResExc`) cuando se intenta registrar un elemento ya existente. Además, se pueden asociar ingredientes a los platos mediante `nuevoingPlato`, calculando sus precios de forma automática con el método `calcularPrecio`, que combina los costos unitarios de los ingredientes y sus cantidades.

La clase también incorpora un módulo de **gestión de órdenes**, con validaciones de inventario (`verificarOrden`) que garantizan que las solicitudes no superen los mínimos requeridos. Si el inventario es insuficiente, el sistema lanza una excepción de restricción, asegurando la trazabilidad de los recursos. El método `agregarOrden` registra pedidos considerando la disponibilidad y el tipo de plato (diario o a la carta), actualizando los valores de venta y la cantidad de ingredientes utilizada.

Finalmente, se implementan funciones de **análisis estadístico y de rendimiento**, como `platosmasSolicitados`, `platosmasRentables` y `totalVentas`, que permiten conocer los tres platos más pedidos, los más rentables y el total de ventas en una fecha específica. Estos métodos ordenan los resultados mediante algoritmos de burbuja y generan reportes textuales legibles para el usuario.

En conjunto, este proyecto constituye una aplicación completa de los fundamentos de **Programación Orientada a Objetos (POO)** en Java, abarcando herencia, polimorfismo, manejo de excepciones, colecciones genéricas y control lógico de inventarios, con un enfoque funcional en la administración eficiente de un restaurante.
